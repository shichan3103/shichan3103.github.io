2、工作目录简介

       很多小可爱应该和我一样，看着自己搭好的博客一头雾水——这在哪写自己的博文呐！为了解决这一疑惑，我们现在来介绍一下工作空间文件夹的目录。

blogsWorkSpace：这是我们开始创建的工作空间（名字是自己起的）

public：三连1 或者二连2 生成的缓存文件（$ hexo clean是删除$ hexo g是生成）。

source：页面和博文存放的地方。

_posts： ！！看到没有你的博文就写在这个文件夹里面！！

_drafts：存放草稿的地方

其他：存放各个页面，如about就是存放关于页面，categories就是分类页面。

themes：主题存放的地方。

_config.yml ：这个是博客的主配置文件。

工作空间的其他目录不做介绍，一定不是因为我也不知道是干嘛的。

二、.md 文件介绍

1、.md 文件简介

       按进入 source 中的 _posts 文件夹，我们可以看到我们刚刚创建的博客文件，均为 .md 文件。.md 文件是由 Markdown 语言编写的文件，Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档。

       所以我们可以通过查询 Markdown 的教程来学习博客的编写方式，但要注意的是有少数 Markdown 的功能在博客中是无效的，后面我会把我发现的功能一一列出（ 附上 Markdown在线编译器 ）。

3、编写工具

       .md 文件的编写门槛其实很低，用电脑自带的记事本、EditPuls 等其他记事本或者各种轻量级编译器都可以。这里向大家推荐一款很好用的编写软件：Typora 。具体使用方法我会和标签插件一起总结一下，以便以后忘了的时候查阅。（ 附上一篇 Typora快捷键教程 ）

       Typora 上方视图菜单中有 专注模式 和 打字机模式 。其中专注模式是只有当前打字的那一行是亮着的，其他地方变成灰暗的（ 个人感觉有点鸡肋 ）。打字机模式是让我们正在打字的这一行始终在页面最中间（ 感觉这个不错，写文章时可以一直开着，不用特意翻页面 ）。

2、文字和图片

       博客添加文字很简单，正常打字便可以。添加图片分为两种，一是网络上的图片，只要依次点击 右键-插入-图片 小括号 () 中填写图片链接就可以；另一种就是本地图片了，本地图片需要按以下步骤进行：

  • 初次导入需下载配置图片上传插件：

    a、将主配置文件 _config.yml 中的 post_asset_folder: false 改为 post_asset_folder: true

    b、在 git 中执行 npm install https://github.com/CodeFalling/hexo-asset-image -- save 以安装插件

  • 插件配置好后添加图片需要两个步骤：

    a、把图片导入 source/_posts/*** 文件夹中（ ***为与博文同名的文件夹 ）如果没有就手动创建。

    b、依次点击 右键-插入-图片 小括号 () 中填写 图片名字.图片格式 ( 由于是博客的插件，Typora无法识别这个图片，不过没关系，上传上去是可以用的 )

       文字的话设置字体、颜色、大小、背景当然也是必要的，以下代码和样式便可以实现这一功能（ 选取颜色代码可在 工具：颜色名列表 界面）
