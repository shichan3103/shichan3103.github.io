2、工作目录简介

       很多小可爱应该和我一样，看着自己搭好的博客一头雾水——这在哪写自己的博文呐！为了解决这一疑惑，我们现在来介绍一下工作空间文件夹的目录。

blogsWorkSpace：这是我们开始创建的工作空间（名字是自己起的）

public：三连1 或者二连2 生成的缓存文件（$ hexo clean是删除$ hexo g是生成）。

source：页面和博文存放的地方。

_posts： ！！看到没有你的博文就写在这个文件夹里面！！

_drafts：存放草稿的地方

其他：存放各个页面，如about就是存放关于页面，categories就是分类页面。

themes：主题存放的地方。

_config.yml ：这个是博客的主配置文件。

工作空间的其他目录不做介绍，一定不是因为我也不知道是干嘛的。

二、.md 文件介绍

1、.md 文件简介

       按进入 source 中的 _posts 文件夹，我们可以看到我们刚刚创建的博客文件，均为 .md 文件。.md 文件是由 Markdown 语言编写的文件，Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档。

       所以我们可以通过查询 Markdown 的教程来学习博客的编写方式，但要注意的是有少数 Markdown 的功能在博客中是无效的，后面我会把我发现的功能一一列出。

2、.md 结构介绍

       用记事本打开 .md 文件如下图（ 我用的记事本是Editplus，大家没有的可以先用普通记事本，后面我会给大家介绍编写博客的工具）：.md 文件由两部分构成——Front-matter 部分，下面不知道叫什么部分。

a、Front-matter 部分

       上两个 --- 分隔的区域就是 Front-matter 部分，可以把它看做是文章的配置部分， 以下是预先定义的参数，我们可以在这个部分中使用这些参数值并加以利用。

参数 描述 默认值
layout 布局
title 标题 文章的文件名
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于分页）
categories 分类（不适用于分页）
permalink 覆盖文章网址
keywords 仅用于 meta 标签和 Open Graph 的关键词（不推荐使用）

       这一部分其实就是我们写文章的部分，我们可以在这里编写我们博文的具体内容，当然也可以用一些标签插件提高我们博文的可读性与美观性。

       很多小可爱应该发现，别人的博文内容明明那么多可是在首页中只显示前几段，下面的用一个 阅读原文 按钮代替了，这是怎么实现的呢？其实很简单，只要在文章的中间加一行 <!-- more --> 在首页是就只显示 <!-- more --> 前面的文章，全文只有在进入博文后才能看到。

3、编写工具

       .md 文件的编写门槛其实很低，用电脑自带的记事本、EditPuls 等其他记事本或者各种轻量级编译器都可以。这里向大家推荐一款很好用的编写软件：Typora 。具体使用方法我会和标签插件一起总结一下，以便以后忘了的时候查阅。

三、博文的编写

       为了方便以后查阅，这里我只总结了在 Typora 软件中各个标签最快捷的使用方法，如果想要学习详细教程，大家可以去 官方标签插件教程 、@ yearito 大大的 教科书般的教程 、介绍 Markdown 的 菜鸟教程 或百度中查询学习。

       Typora 上方视图菜单中有 专注模式 和 打字机模式 。其中专注模式是只有当前打字的那一行是亮着的，其他地方变成灰暗的（ 个人感觉有点鸡肋 ）。打字机模式是让我们正在打字的这一行始终在页面最中间（ 感觉这个不错，写文章时可以一直开着，不用特意翻页面 ）。
